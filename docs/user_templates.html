»User Templates

Overview

Users are now allowed 3 templates to edit. They are:

  1. Blog Template
  2. Friends Template
  3. Profile Template
Each controls a different aspect of the users template. There is little difference between the Blog template and the Friends template. All three do share some common tags.

NLB 3's templates are powered by ETS (Easy Template System). Every user template is called by the {safe} function. This means that users may not use the {php} or {if: } features of ETS. They are disabled for security reasons.

About Tags & Masks

Those of you familiar with the template syntax of NLB 2, there is nothing different in version 3.

The basic tag has this syntax: {tag}. An example tag is {username}. You can put this tag in your template and it will be replaced with your username.

Masks are how ETS loops throug tags. A common mask looks like this: 

{mask:blog}
...
{tag}
...
{/mask}

{mask:blog} contains your blogs. As your blogs are printed out, {tag} will be replaced with the value for that specific blog in the loop.

»Common Tags

These tags exist in all three template files. Tags marked with a * may not always have a value.

Tag Value
{username} Username of template owner
{url_home} URL to users blog page. Use with <a>
{url_profile} URL to users profile page.
{url_friends} URL to users friends page.
{rss_url} URL to users RSS feed.
{rss_img} A click-able image to users RSS feed
{avatar} * A <img> tag that displays users avatar.
{avatar_url} * URL to users avatar
{url_page_next} * URL to next page of blogs (blogs & friends template only)
{url_page_prev} * URL to the previous page of blogs (blog & friends template only)

»Blog Template

Mask: Blog

These tags are to be used with {mask:blog}. Tags with a * may not always have a value.

Tag Value
{subject} * The subject of the blog
{date} Date and time blog was posted
{body} The body of the blog
{mood} * The mood of the user
{custom_title} * The name of the custom field chosen by the user
{custom_text} * Value for custom field
{comments} If {comments} has a value, then this user allows the public to post comments
{comments_url} URL to view/post comments for this blog
{comment_num} The number of comments
{author} Author of blog
{author_url_profile} URL to authors profile
{author_url_blogs} URL to authors blog page

Mask: list_comments

Tags inside {mask:list_comments}.
Imporant: use {set:view_comments} to see if comments need to be displayed.

Tag Value
{author} Username of the author of the blog
{guest} Use this with {set} to decide if this is an Anonymous comment.
{author_url_blog} * URL to the authors blogs
{author_url_profile} * URL to the authors profile
{body} Comment body
{avatar_url} * URL to comment authors avatar, if one is set
{avatar} * A <img> tag that dispays authors avatar, if one is set

»Friends Template

Mask: Blog

Tags used in {mask:blog} are identical to those in the normal blog template.

Mask: friend_list

{mask:friends_list} will let you display a list of your friends.

Tag Value
{name} Username of friend
{url_blogs} URL to friends blogs
{url_profile} URL to friends profile
